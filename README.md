# バドミントンランク戦管理スクリプト

## スクリプト一覧
- `rank_match_script` ランク戦の日程報告と結果報告フォーム、日程と順位表シートを扱うスクリプト
- `rank_result_check_form_script` ランク戦の個人成績を確認するためのフォームを扱うスクリプト

## 日程報告フォーム

スクリプト `rank_match_script`

### 機能一覧

- 日程追加機能

以下のように入力する。

| ランク戦を申し込む人 | ランク戦を受ける人 | ランク戦の日付 | 時間帯 | キャンセル | 変更後の日付 | 変更後の時間帯 |
|:-----------:|:------------:|:------------:|:-----------:|:------------:|:-----------:|:------------:|
|自分の名前|対戦相手の名前|yyyy/mm/dd|金曜部活内/部活時間外/その他|空欄|空欄|空欄|

- 日程削除機能

以下のように入力する。

| ランク戦を申し込む人 | ランク戦を受ける人 | ランク戦の日付 | 時間帯 | キャンセル | 変更後の日付 | 変更後の時間帯 |
|:-----------:|:------------:|:------------:|:-----------:|:------------:|:-----------:|:------------:|
|自分の名前|対戦相手の名前|yyyy/mm/dd|金曜部活内/部活時間外/その他|チェック入れる|空欄|空欄|

キャンセルしたい試合の情報を入れ、キャンセルマークにチェックを入れる。

- 日程変更機能

以下のように入力する。

| ランク戦を申し込む人 | ランク戦を受ける人 | ランク戦の日付 | 時間帯 | キャンセル | 変更後の日付 | 変更後の時間帯 |
|:-----------:|:------------:|:------------:|:-----------:|:------------:|:-----------:|:------------:|
|自分の名前|対戦相手の名前|yyyy/mm/dd|金曜部活内/部活時間外/その他|空欄|yyyy/mm/dd|金曜部活内/部活時間外/その他|

日程変更したい試合の情報を入れ、修正後の日付、時間帯を入力する。修正後の日付、時間帯の両方が入力されていなければならない。

### 制約

以下の事項に当てはまるフォームはすべて無効となる。
- 同一人物が対戦する。
- 申込者と対戦相手の性別が異なる。
- 申込者のランクの方が対戦相手のランクよりも高い。
- 申込者と対戦相手のランク差が規則で定められた差よりも大きい。
- 試合予定日が過去のものである。
- 試合予定日が規則で定められた日数よりも未来である。
- 時間帯が金曜部活内であるにも関わらず、試合予定日が金曜ではない。
- 既に金曜部活内の最大試合可能数に達しているにも関わらず、追加の金曜部活内の試合を登録する。
- 申込時点で試合可能でない人が試合しようとしている。
- 挑戦回数が0回になっているにも関わらず試合の申込者になっている。
- 削除や日程変更をしようとしている試合が存在しない。
- 過去の試合の削除や日程変更をしようとしている。
- 日程変更の際、変更前と後で日付と時間帯が変わっていない。
- 既に日程変更を行なった試合を再び日程変更しようとしている。
- 変更後の日付と変更後の時間帯のうち片方のみが入力されている。
- 同じ対戦カードを規則で定められたクールタイムを待たずに行おうとしている（詳細は後述）。

また、上記に当てはまっていなくても以下の場合、フォームは無効となる場合が存在する。
- スクリプト内部でエラーが発生した。
- フォームの提出が立て込み、処理しきれなかった。

無効となった場合、シートのログのセルにその理由が記載される。
正常に処理が終了した場合は何も記入されない。
もし無効になっても特に報告はいらず、もう一度フォームを送り直せば良い。

**ただし、以下の場合は直ちに管理者にその旨を報告すること。**

1. フォームが無効となった際、以下のようなログがセルに記入された。
- `日程報告中に予期せぬエラーが発生しました。`
- `日程追加処理中にエラーが発生しました。`
- `キャンセル処理中にエラーが発生しました。`
- `日程/時間帯変更処理中にエラーが発生しました。`

2. シートに意図しない変更を行った、または覚えのない試合が組まれている。

3. 規定している処理ではない処理を行っているなどの不具合を発見した。

## 結果報告フォーム

スクリプト `rank_match_script`

### 使い方

1. 結果を報告したい試合の情報を入力する。
2. 申込者から見た勝敗を選択する。（勝利/敗北/不戦勝）
3. もし、スコアも覚えていて記入できる際は記入する。

### 制約

- 存在しない試合の結果報告は受け付けない。
- 当日の試合のみの結果報告しか受け付けない。（もしその日中に提出が確認できなかった場合、その試合は無効になる。挑戦回数は回復しない。）
- 既に結果報告を受けた試合の結果報告は受け付けない。
- 結果報告時点で既に申込者のランクが対戦相手よりも高い場合、結果は記録するがその試合による順位変動は発生しない。
- 挑戦回数が0となっている人でもその月の申し込んだ試合にすべて勝利した場合、結果報告後に挑戦回数が1回復する。これは申し込んだ試合で敗北するまで継続する。
- 勝利報告による順位変動は以下の法則に従い決定される。
    1. 申込日から試合を行うまでに対戦相手が別の試合などで勝利していない場合
        - 申込者と対戦相手のランク差に関わらず申込者が対戦相手のランクになる。

    2. 申込日から試合を行うまでに対戦相手が別の試合などで勝利している場合
        - ランク差と規則で定められた対戦申込可能なランク差のうち、小さい方の分申込者のランクが上がる。

## 結果確認フォーム

スクリプト `rank_result_check_form_script`

### 使い方

1. 結果確認フォームで結果を参照したい人を選ぶ
2. もし特定のカードについての結果を参照したい場合はその対戦相手も選ぶ
3. フォームを送信すると指定したgmailに対戦結果を含むメールが送信される

### 返信フォーマット

- **一人のみを指定した時**

結果が存在する場合
```
以下、 山田太郎 (XXXXXXXX) さんの過去のランク戦結果です。

2026/4/16 勝利  山田太郎 (XXXXXXXX) vs  仙台四郎 (YYYYYYYY) 13-15 15-13 15-10
2026/4/22 敗北  田中三郎 (ZZZZZZZZ) vs  山田太郎 (XXXXXXXX) 15-10 15-13

1勝1敗（勝率：50.00％）
```

結果が存在しない場合
```
佐藤花子 (VVVVVVVV) さんの試合は存在しませんでした。
```

- **対戦相手も指定した時**

結果が存在する場合
```
以下、 山田太郎 (XXXXXXXX) さん 対  田中三郎 (ZZZZZZZZ) さんの過去のランク戦結果です。

2026/4/22 敗北  田中三郎 (ZZZZZZZZ) vs  山田太郎 (XXXXXXXX) 15-10 15-13

0勝1敗（勝率：0.00％）
```

結果が存在しない場合
```
佐藤花子 (VVVVVVVV) さん 対  加藤花子 (UUUUUUUU) さんの試合は存在しませんでした。
```

### 制約

- 結果の各行は以下の順で出力される

| 対戦日 | `使い方 1.`で指定した人から見た勝敗 | その試合の申込者 | 対戦相手 | スコア（未入力の場合空白）|
|:-----------:|:------------:|:------------:|:-----------:|:------------:|
|yyyy/mm/dd|勝利/敗北/不戦勝/不戦敗|名前 (ID)|名前 (ID)|x-x|

- 何らかのエラーが発生し、メールが送信されなかった場合、そのことをフォーム利用者は**分からない**。
エラーログはスクリプトの実行ログに残るため参照権限を持つ者が調査すること。
